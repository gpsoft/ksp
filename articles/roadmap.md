# プログラマーになるためのロードマップ

プログラムの中でも、アプリケーション・プログラムを対象にする。

個々のプログラミング言語に依存する話しは、除く。

## アプリケーション・プログラムの種類

プログラムの種類によって、違いがありそう。

- CLI、コンソールアプリ
- デスクトップアプリ
- モバイルアプリ
- Webアプリ

## 共通

どんなプログラマにも共通する要素。

### ツールの使いこなし

- パソコン、OS(Windows, Mac, Linux)
- テキストエディタ
- ターミナル、シェル
- Git, GitHub
- Docker
- Virtual Box

### 理論、概念、原理の理解

- コンピュータの仕組み、プログラムが動く仕組み
  - CPU、レジスタ
  - ROM
  - RAM
  - I/O
  - アセンブラ

- OSの仕組み
  - プロセス
  - スレッド
  - 非同期処理、割り込み処理
  - ファイルシステム

- ネットワークの仕組み
  - IPアドレス
  - socket, port
  - TCP, UDP
  - ホスト名、ドメイン名、DNS
  - Macアドレス

- 文字コード
  - 文字セット
  - 符号化方式

- データ構造(とアルゴリズムをほんの少し)
- 正規表現
- 暗号、PKI
- オブジェクト指向
- 関数型プログラミング
- MVC
- デザインパターン
- 外部依存
  - ライブラリ
  - Web API
- ビルド管理、パッケージ管理
- データ記法
  - JSON
  - YAML
  - XML
- デバッグ
  - printfデバッグ
  - ブレークポイント
  - スタックトレース
  - ロギング
  - テストの自動化
- キャッシュ

### 実技、経験、その他

- 英語
  - 読解

  - リスニング(YouTube)

- タッチタイピング

## プログラミング言語の習得

どのプログラミング言語を学ぶにしろ、共通するのは4つの柱がある、という点。

- 文法を学ぶ
- ツールを使いこなす
- ライブラリの使い方を学ぶ
- 設計スタイルやイディオムを学ぶ
